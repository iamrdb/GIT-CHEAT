GIT COMMAND – CHEAT SHEET

Developed By : Rahul Deb Bhandari

It is a open source distributed version control system to handle projects. It have three area : Working Area , Staging Area , Local Repository Area.

CREATE REPOSITORY

git config –-global user.name [username] : Configuration of usernameglobally. git config –-global user.name : To show configured username. git config –-global user.email[email address] : Configuration of user email addressglobally. git init : To initialize git folder into working directory.

SYNC REPOSITORY

git remote add origin [repository link] : Link a remote repositorywith local repository. git pull origin master : Copy all the files from of remote repositorytothe master branch of local repository. git push origin master : Copy all the files from the master branch of local repository to remote repository git fetch [repository link] [branch] : Fetches a specific , from the repository. Leave off to fetch all remote refs.

MAKE CHANGES

git status : To show current status of repository. git status –s : To show current status of repository in short. git add [filename] : To add given untracked file of working directory into staging area. git add –A / git add –-all / git add . : To add all untracked files of working directory into staging area. git mv [old file name] [new file name] : To rename file. git checkout [filename] : Roll back to the previous commited content. git checkout –f : Roll back to the previous commited content for all files. git diff : Compare all files of working area with staging area. git diff --staged : Compare staging area with last commit. git diff HEAD : Show difference between working directory and last commit. git diff –cached : Show difference between staged changes and last commit. git reset–[filename] : Remove given file from staging area to working directory. git reset : Reset staging area to match most recent commit, but leave the working directory unchanged. git reset –hard : Reset staging area and working directory to match most recent commit and overwrites all changes in the working directory. git clean -n : Shows which files would be removed from working directory. git clean -f : To execute the clean. git commit –m “[message]” : To make commit with a mesaage. git commits –p -1 : To check last 1 commits. git log : Check all status according to commits. git log [branch name] : Check all status according to commits of a specific branch name. git log –oneline : Check all status according to commits in one line

PARALLEL DEVELOPMENT

git branch : Show list of branch and highlight the branch that we working on gitbrach [branch name] : Create new branch git checkout [branch name] : To switch the branch git merge [branch name] : Merge content or changes of given branch name with present working branch git rebase [base branch name] : Let, we create a branch “b2” with the base of master after that if there are any changes made in master branch which is the base of branch “b2” .So we can update the branch “b2” with the new base of master git branch -d [localBranchName] : Delete branch locally git push [remote name] -d [remoteBranchName] : Delete branch remotely. (Most of the case remote name : origin)

OTHERS

touch [filename] : To make a blank file. rm [filename] : Delete the file from working directory and systems also. git archive [branch name] : To create zip file of given branch name to share with other developer git bundle create: git stash : To save all uncommited changes to temoporarydiretory git ls-files : List of all files on the present working branch ls –lart : To show all hidden file .
